## 전체 구조 개요

### 프로젝트 아키텍처
- **프레임워크**: React 19.2.0 + TypeScript
- **빌드 도구**: Vite 5.4.20
- **스타일링**: CSS Modules + 전역 CSS 변수
- **상태 관리**: Zustand
- **UI 라이브러리**: React Icons, TipTap (리치 텍스트 에디터)

### 디렉토리 구조
```
src/
├── components/          # 재사용 가능한 컴포넌트
│   ├── layout/         # 레이아웃 컴포넌트 (Header, Sidebar)
│   ├── features/       # 기능별 컴포넌트 (각 페이지별)
│   └── ui/            # 공통 UI 컴포넌트 (Button, Card, Input)
├── pages/              # 페이지 컴포넌트
├── hooks/              # 커스텀 훅
├── styles/             # 전역 스타일
│   ├── variables.css   # CSS 변수 정의
│   └── pages/          # 페이지별 스타일
└── types/              # TypeScript 타입 정의
```

## 디자인 시스템 분석

### 1. CSS 변수 시스템 (`src/styles/variables.css`)

#### 폰트 시스템
- **메인 폰트**: Paperozi (한글 폰트, 9가지 weight 지원)
- **폴백 폰트**: Apple SD Gothic Neo, Noto Sans KR, Malgun Gothic
- **폰트 크기**: xs(11px) ~ 4xl(32px) - clamp() 사용으로 반응형
- **폰트 굵기**: 400(normal) ~ 800(extrabold)

#### 색상 팔레트
- **사이드바 테마**: 검은색(#1a1a1a) 배경, 흰색 호버
- **Primary 색상**: 테마별로 다름
  - Default: #47CB81 (초록색)
  - Purple: #8b5cf6
  - Green: #10b981
  - Gray: #64748b

#### 간격 시스템
- `--spacing-gap`: clamp(6px, 0.74vh, 10px)
- `--spacing-padding`: clamp(12px, 1.5vw, 20px)
- 반응형 단위: clamp()를 활용한 유연한 크기 조정

#### 테두리 및 그림자
- `--sidebar-radius`: clamp(12px, 1.5vw, 20px)
- `--sidebar-radius-sm`: clamp(10px, 1.25vw, 16px)
- 그림자: 미니멀리즘 스타일 (대부분 none 또는 매우 약한 그림자)

## 레이아웃 구조

### 1. 전체 레이아웃 (`App.tsx`)

```
┌─────────────────────────────────────┐
│  Sidebar (고정)  │  Main Panel      │
│                 │  ┌──────────────┐ │
│  - 로고         │  │ Header        │ │
│  - 메뉴         │  └──────────────┘ │
│  - 로그아웃     │  ┌──────────────┐ │
│                 │  │ Content       │ │
│                 │  │ (스크롤 가능) │ │
│                 │  └──────────────┘ │
│                 │  ┌──────────────┐ │
│                 │  │ Chat (고정)   │ │
│                 │  └──────────────┘ │
└─────────────────────────────────────┘
```

### 2. 사이드바 (`Sidebar.tsx`)
- **위치**: 고정(fixed), 좌측
- **너비**: 
  - 확장: clamp(200px, 12vw, 250px)
  - 축소: clamp(96px, 6vw, 110px)
- **배경색**: #1a1a1a (검은색)
- **기능**: 
  - 로고 클릭으로 확장/축소
  - 메뉴 항목 호버 시 흰색 배경
  - 하위 메뉴는 플라이아웃 방식

### 3. 헤더 (`Header.tsx`)
- **높이**: clamp(56px, 6.5vh, 84px)
- **구성 요소**:
  - Breadcrumb (페이지 섹션 표시)
  - Submenu Tabs (하위 메뉴 탭)
  - 검색창
- **스타일**: 미니멀리즘, 하단 실선 구분

### 4. 메인 패널
- **배경**: 흰색(#ffffff)
- **위치**: 사이드바 오른쪽, 고정
- **내용 영역**: 회색 배경(#f5f5f5)

## 컴포넌트 분석

### 1. 공통 UI 컴포넌트

#### Button (`components/ui/common/Button.tsx`)
- **Variants**: primary, secondary, danger, warning
- **Sizes**: sm, md, lg
- **스타일 특징**:
  - Primary: 검은색 배경 → 호버 시 흰색 배경 + 검은색 텍스트
  - Border-radius: var(--sidebar-radius-sm)
  - Transition: var(--transition-base)

#### Card (`components/ui/common/Card.tsx`)
- **기능**: hoverable, clickable 옵션
- **구조**: header (title/subtitle) + content
- **스타일**: 
  - 배경: var(--card-bg) (#f8f8f7)
  - 테두리: var(--border-color)
  - 호버 시: 위로 2px 이동 + 그림자

#### Input (`components/ui/common/Input.tsx`)
- **타입**: text, email, password, number 등
- **스타일**:
  - Focus 시: 검은색 테두리 + 그림자
  - Placeholder: 회색 텍스트
  - Disabled: 연한 회색 배경

### 2. 레이아웃 컴포넌트

#### Sidebar
- **메뉴 구조**: 
  - 대시보드
  - 공지사항
  - 문서 (하위: 문서관리, 새 문서, 결재관리)
  - 일정 (하위: 학사 일정, 개인 시간표)
  - 학생 및 교직원 (하위: 학생, 교직원)
  - 회계 (조건부)
  - 구글 (하위: 앱스크립트, 구글시트 등)
  - 관리자 (조건부)
- **상태 관리**: 
  - 확장/축소 상태
  - 활성 페이지 추적
  - 로그아웃 모달

#### Header
- **검색 기능**: 페이지별 placeholder 동적 변경
- **네비게이션**: Breadcrumb + Submenu Tabs
- **반응형**: flex-wrap으로 작은 화면 대응

## 스타일링 패턴

### 1. CSS 변수 활용
- 전역 변수: `src/styles/variables.css`
- 테마별 변수: `.theme-default`, `.theme-purple`, `.theme-green`, `.theme-gray`
- 반응형 단위: `clamp()` 함수 활용

### 2. 컴포넌트별 CSS 파일
- 각 컴포넌트마다 별도 CSS 파일
- BEM 스타일 네이밍 (일부)
- 클래스 기반 스타일링

### 3. 반응형 디자인
- `clamp()` 함수로 최소/선호/최대 크기 지정
- Viewport 단위(vw, vh) 활용
- 미디어 쿼리 최소화 (clamp로 대부분 해결)

### 4. 전역 스타일
- `index.css`: 전역 리셋, 폰트 로딩
- 스크롤바 숨김 처리
- 텍스트 선택 활성화

## 개선 제안

### 1. 디자인 시스템 통합
**현재 문제점**:
- CSS 변수가 여러 파일에 분산
- 일부 컴포넌트가 하드코딩된 색상 사용
- 테마 전환이 완전히 일관되지 않음

**개선 방안**:
- 모든 색상을 CSS 변수로 통일
- 디자인 토큰 문서화
- 테마 시스템 강화

### 2. 컴포넌트 재사용성
**현재 문제점**:
- 유사한 스타일이 여러 파일에 중복
- 일관성 없는 버튼/카드 스타일

**개선 방안**:
- 공통 컴포넌트 확대 사용
- 스타일 가이드 문서화
- Storybook 도입 고려

### 3. 반응형 디자인
**현재 상태**:
- clamp() 활용으로 기본적인 반응형 구현

**개선 방안**:
- 모바일 레이아웃 추가
- 태블릿 중간 크기 최적화
- 터치 인터랙션 개선

### 4. 접근성
**개선 필요 사항**:

### 5. 성능 최적화
**개선 방안**:
